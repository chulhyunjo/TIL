DB 트랜잭션

- DB에서 무결성을 유지하기 위해 특정 작업이 모두 수행되거나 모두 수행되지 않게 하는 개념
- BEGIN: 트랜잭션이 시작
- COMMIT: 트랜잭션 시작 이후 수행된 모든 연산을 DB에 적용
- ROLLBACK: 트랜잭션 시작 이후 수행된 모든 연산을 취소

```sql
DROP TABLE zoo;
CREATE TABLE zoo (
  name TEXT NOT NULL,
  eat TEXT NOT NULL,
  weight INT NOT NULL,
  height INT,
  age INT
);
delete from zoo;
INSERT INTO zoo VALUES 
('gorilla', 'omnivore', 215, 180, 5),
('tiger', 'carnivore', 220, 115, 3),
('elephant', 'herbivore', 3800, 280, 10),
('dog', 'omnivore', 8, 20, 1),
('panda', 'herbivore', 80, 90, 2),
('pig', 'omnivore', 70, 45, 5);
BEGIN;
  DELETE FROM zoo
  WHERE weight < 100;
ROLLBACK;
BEGIN;
  DELETE FROM zoo
  WHERE eat = 'omnivore';
COMMIT;
SELECT * FROM zoo;

```
